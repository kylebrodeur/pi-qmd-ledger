│                                                                              │
│ *   **The Scalability Challenge:** Research on the Human-in-the-loop         │
│ Multi-Robot Collaboration Framework (HMCF) notes that while multi-agent      │
│ architectures perform well in small scenarios, scalability remains a primary │
│ challenge for large deployments involving dozens or hundreds of robots [1,   │

│ *   **Cloud-Inference Bottlenecks:** A major contributor to this             │

│ back-and-forth communication between local robotic systems and cloud-hosted  │
│ LLM agents is heavily constrained by signal transmission delays, making      │
│ reliable connectivity a significant obstacle for large-scale deployment [1,  │
│ 2].                                                                          │
│ *   **Information Overload and Hallucinations:** As the amount of            │
│ communication and the number of interacting agents increase within a shared  │
│ network, LLM agents face severe information overload, which directly leads   │
│ to an increase in AI hallucinations [3, 4].                                  │
│ *   **Modeling Constraints:** When attempting to simulate these systems,     │
│ Agent-Based Modeling (ABM) frameworks are similarly noted as being "complex  │
│ for very large agent populations," reflecting the inherent difficulty of     │
│ managing massive numbers of interacting agents [5].                          │
│                                                                              │
│ **Architectural Mitigations for Communication Overhead**                     │
│ To manage the communication load as systems scale, researchers are           │
│ implementing specific structural mechanisms:                                 │
│ *   **Targeted Messaging (The "@" Function):** To prevent agents from being  │
│ overwhelmed by global network traffic, systems like HMCF utilize an "@"      │
│ tagging function. While a human supervisor can see the entire conversation,  │
│ each robot's LLM agent only processes the data specifically tagged with its  │

│ mitigating hallucinations [3, 4].                                            │
│ *   **Shared Persistent Memory and Asynchronous Execution:** Frameworks like │
│ CORAL replace constant, rigid control loops with asynchronous execution and  │

│ *   **Heartbeat Interventions:** To prevent agents from entering infinite    │
│ communication loops or consuming excess compute resources during             │
│ long-horizon tasks, systems utilize "heartbeat-based interventions" to       │
│ actively manage agent health and resource consumption [6, 7].                │

│ Based on the provided research, while there is no single universal "scaling  │
│ law" explicitly named for all Multi-Agent Systems (MAS), empirical studies   │
│ on distributed, collaborative industrial networks (such as federated         │
│ learning systems) reveal clear scaling behaviors that dictate when           │
│ communication overhead overtakes computation.                                │
│                                                                              │
│ **Scaling Behaviors for Communication vs. Computation**                      │
│ In distributed agent networks, the scaling of communication and computation  │
│ follows divergent paths:                                                     │
│ *   **Communication Overhead Scales Linearly:** The communication overhead   │
│ required to share updates between nodes scales **approximately linearly**    │
│ with both the number of participating agents (or facilities) and the size of │
│ the models being shared [1]. For example, in an industrial network using a   │
│ 478 MB base model, every additional facility added to the federation         │
│ increased the communication overhead by a predictable 23.8 MB per round [1]. │
│ *   **Computation Scales Sub-Linearly:** Conversely, the actual              │
│ computational training or processing time per round exhibits **sub-linear    │
│ scaling** [1]. Because distributed agents process their local tasks in       │
│ parallel, the computational efficiency actually improves as the network      │
│ grows [1].                                                                   │
│                                                                              │
│ **When Messaging Dominates Computation**                                     │
│ Because communication overhead grows linearly while computation time grows   │
│ sub-linearly, **messaging begins to dominate computation as the network size │
│ (number of agents) or the model complexity scales up** [1]. In these         │
│ large-scale or highly complex environments, the bandwidth required to        │
│ transmit data overtakes the time required to process it locally, especially  │
│ in environments with limited network bandwidth [2, 3].                       │
│                                                                              │
│ In modern, LLM-driven Multi-Agent Systems, this dynamic is further amplified │
│ by how agents interact:                                                      │
│ *   **Structured Conversation Overhead:** Unlike classical symbolic agents   │
│ that use rigid, lightweight algorithmic protocols (like the Contract Net     │
│ Protocol), neural MAS coordinate through "structured conversation and        │
│ prompt-driven orchestration" [4].                                            │
│ *   **Context and Token Limits:** A central orchestrator agent manages the   │
│ dynamic workflow through structured messaging [5]. As agents converse, the   │
│ shared context window grows, meaning the system's primary practical          │
│ constraints become the latency and computational expense of processing these │
│ messages (measured in total tokens and API calls) over extended interactions │
│ [6].                                                                         │
│                                                                              │
│ Therefore, messaging dominates computation in MAS when the network of agents │
│ becomes very large, when the models or context windows being exchanged are   │
│ highly complex, or when the system relies on continuous, token-heavy         │
│ conversational coordination rather than explicit, localized rule execution   │
│ [1, 4, 6].                                                                   │
