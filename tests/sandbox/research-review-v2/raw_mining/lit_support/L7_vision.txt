│ **The Role of Qwen in Task Decomposition and Feasibility Assessment**        │
│ To make complex, long-horizon robotic manipulation feasible, tasks must be   │
│ broken down into achievable, verifiable milestones [4, 5]. Advanced VLMs     │
│ like **Qwen3 and Qwen2.5-VL** are actively utilized to automate this         │
│ trajectory decomposition [3]:                                                │
│ *   **Qwen3** is used to cluster instruction verbs into a structured library │
│ of atomic robotic skills, such as "pick" or "push" [3].                      │
│ *   **Qwen2.5-VL (72B)** is then deployed to analyze physical state changes  │
│ in the robot's trajectory, merging adjacent segments that share identical    │
│ skills and autonomously generating precise, natural language subtask         │
│ descriptions [3].                                                            │
│                                                                              │
│ This VLM-driven abstraction converts continuous, ambiguous tasks into a      │
│ sequence of discrete, structured milestones [5]. It provides the policy with │
│ explicit references for "what" to execute and "how" to verify that a task    │
│ stage has been successfully completed, ensuring that tasks remain            │
│ practically feasible for the robot [5].                                      │
│                                                                              │
│ **Reducing Trajectory and Execution Errors**                                 │
│ Standard VLA models attempt to map visual observations and linguistic        │
│ instructions directly to low-level robotic actions [2]. However, these       │
│ models are notoriously brittle and prone to severe execution errors when     │
│ faced with novel, out-of-distribution objects [2]. This occurs because       │
│ linguistic sub-goals, while semantically useful, lack the concrete spatial   │
│ and physical constraints required to generate an accurate physical           │
│ trajectory [6, 7]. For instance, when commanded to grasp an unseen soda      │
│ bottle, a standard VLA might interpolate actions from a training object      │
│ (like a can) and incorrectly attempt a horizontal grasp approach rather than │
│ a vertical one, resulting in task failure [7].                               │
│                                                                              │
│ To eliminate these physical trajectory and execution errors, modern          │
│ architectures like VISTA (VIsual Subgoal TAsk decomposition) utilize world   │
│ models as high-level planners to synthesize **visually grounded goal         │
│ images** alongside textual instructions [8, 9].                              │
│ *   **Explicit Spatial Cues:** These generated visual goals act as spatial   │
│ blueprints, explicitly encoding the required interaction region, the correct │
│ approach trajectory, and the desired end-effector posture [10].              │
│ *   **Multi-View Consistency:** The models ensure that the generated goals   │
│ maintain spatial coherence across multiple camera angles [11, 12]. This      │
│ prevents spatial misalignment that would otherwise cause the robotic gripper │
│ to adopt physically impossible postures or collide with objects during       │
│ execution [13].                                                              │
│                                                                              │
│ By conditioning the robot's low-level controller on these VLM-assisted       │
│ visual subgoals, the system is provided with strict spatial constraints that │
│ language alone cannot convey [12, 14]. This drastically reduces              │
│ manipulation, trajectory, and grasping errors when encountering new          │
│ environments or objects [15, 16]. In real-world testing, guiding robotic     │
│ policies with these visually generated milestones boosted the execution      │
│ success rate in novel, unconstrained scenarios from a mere 14% to 69% [15].  │
