│ In decentralized multi-agent systems and post-platform economies,            │
│ traditional inflationary and deflationary dynamics are replaced by           │
│ community-managed mutual credit mechanisms. Projects like Holochain and the  │
│ MetaCurrency Project have designed non-monetary, reputation-based currencies │
│ (termed "current-sees") to track flows of value, services, and community     │
│ metrics such as timeliness, reliability, and peer feedback [1, 2].           │
│                                                                              │
│ The dynamics of these reputation-based currencies operate on the following   │
│ principles:                                                                  │
│                                                                              │
│ *   **Zero-Balance Mutual Credit:** Unlike traditional fiat currencies where │
│ a central bank controls the money supply, these systems operate on a mutual  │
│ credit basis where all accounts begin with a zero balance [3]. Currency is   │
│ only created when a transaction occurs and one participant extends credit to │
│ another (for example, one account is debited -20 credits while the other is  │
│ credited +20), ensuring that the overall system balance always remains at    │
│ exactly zero [3].                                                            │
│ *   **Usage-Driven Expansion and Contraction:** Because there is no          │
│ centralized limit on the number of assets in the system, the expansion       │
│ (inflation) and contraction (deflation) of the currency supply are           │
│ determined organically by the actual usage patterns and transaction          │
│ histories of the community [3, 4].                                           │
│ *   **Reputation-Governed Credit Limits:** To manage this dynamic supply,    │
│ the community establishes individual "credit limits," which dictate the      │
│ maximum negative balance a user is allowed to hold [4]. These limits are     │
│ dynamically calculated based on a participant's transaction history and      │
│ reputation within the network [4].                                           │
│                                                                              │
│ Consequently, **inflation and deflation are directly tied to community       │
│ activity and reputation rather than central monetary policy**. A             │
│ participant's reputation metric governs their credit limit, which in turn    │
│ safely regulates the expansion and contraction of the currency supply within │
│ the decentralized ecosystem [2, 4].                                          │
